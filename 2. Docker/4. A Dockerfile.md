# Dockerfile

## What is a Dockerfile?

A Dockerfile is a text file that contains a set of instructions used to automatically build a Docker Image. It defines the application's environment, dependencies, configuration, and the commands required to run the application inside a container.

Instead of manually configuring a container each time, a Dockerfile allows you to automate the image creation process, ensuring consistency and repeatability.

---

## Why Use a Dockerfile?

- Automates Docker image creation
- Ensures consistent application environments
- Eliminates manual configuration
- Makes application deployment reproducible
- Simplifies collaboration across teams

---

## Dockerfile Workflow

```text
+-------------+      docker build       +--------------+      docker run       +------------------+
| Dockerfile  |  -------------------->  | Docker Image |  ------------------>  | Docker Container |
+-------------+                         +--------------+                       +------------------+
```

## Basic Dockerfile Example

```dockerfile
FROM python:3.12-slim

WORKDIR /app

COPY . .

RUN pip install flask

CMD ["python", "app.py"]
```

---

| Instruction | Purpose | Example |
|-------------|---------|---------|
| `FROM` | Specifies the base image for the Docker image. | `FROM python:3.12-slim` |
| `WORKDIR` | Sets the working directory inside the container. | `WORKDIR /app` |
| `COPY` | Copies files and directories from the host to the container. | `COPY . .` |
| `ADD` | Copies files, extracts local archives, and supports downloading from URLs. | `ADD app.tar.gz /app/` |
| `RUN` | Executes commands during the image build process. | `RUN pip install flask` |
| `ENV` | Sets environment variables inside the container. | `ENV APP_ENV=production` |
| `EXPOSE` | Documents the port the application listens on inside the container. | `EXPOSE 5000` |
| `CMD` | Specifies the default command to execute when the container starts. | `CMD ["python", "app.py"]` |
| `ENTRYPOINT` | Defines the main executable that always runs when the container starts. | `ENTRYPOINT ["python"]` |

---

## CMD vs ENTRYPOINT

| Feature | CMD | ENTRYPOINT |
|---------|-----|------------|
| What is it? | Specifies the default command to run. | Specifies the main command that always runs. |
| Can it be overridden? | ✅ Yes, easily overridden when running the container. | ❌ No, only its arguments can be changed by default. |
| Main Purpose | Provides a default command or arguments. | Defines the container's executable. |
| Best Use Case | When users may want to run a different command. | When the container should always run the same application. |
| Example | `CMD ["python", "app.py"]` | `ENTRYPOINT ["python"]` |
| Runtime Example | `docker run app python test.py` → Runs `python test.py` | `docker run app test.py` → Runs `python test.py` because `python` is fixed as the ENTRYPOINT. |

- **CMD** → Default command (can be replaced).
- **ENTRYPOINT** → Fixed command (always runs).
- **ENTRYPOINT + CMD** → Best practice. `ENTRYPOINT` defines the executable, and `CMD` provides its default arguments.

## Build an Image Using Dockerfile

```bash
docker build -t myapp .
```

---

## Run the Image

```bash
docker run -d -p 5000:5000 myapp
```

---

## Key Point

- A Dockerfile is the blueprint for creating a Docker Image, while the Docker Image is used to create one or more Docker Containers.
