# Docker Architecture Explained

Docker follows a **Client-Server Architecture** where the Docker Client sends commands to the Docker Engine, which manages Docker resources such as Images, Containers, Networks, and Volumes.

## 1. Docker Client

This is where users interact with Docker using commands.

Examples:

```bash
docker build
docker pull
docker run
docker ps
docker stop
```

The Docker Client sends these commands to the Docker Engine.

---

## 2. Docker Engine

Docker Engine is the core component of Docker that processes commands and manages containers.

It consists of:

### Docker Daemon (dockerd)

The background service responsible for:

- Building images
- Creating containers
- Managing networks
- Managing volumes

### REST API

Provides an interface through which Docker Client communicates with Docker Engine.

### Docker CLI

Command-line tool used by users to interact with Docker.

---

## 3. Docker Registry

A Docker Registry stores Docker Images.

The Docker Engine can:

- Pull images from a registry
- Push images to a registry

Examples:

- Docker Hub
- Amazon ECR
- Google Artifact Registry

Example:

```bash
docker pull nginx
docker push myimage
```

---

## 4. Docker Objects

Docker Engine manages four main objects:

### Images

Read-only templates used to create containers.

Example:

```bash
docker pull ubuntu
```


### Containers

Running instances of Docker Images.

Example:

```bash
docker run ubuntu
```


### Networks

Allow communication between containers.

Example:

```bash
docker network create mynetwork
```

### Volumes

Provide persistent storage for containers.

Example:

```bash
docker volume create myvolume
```

---

# Complete Workflow

```text
User
  │
  ▼
Docker Client
  │
  ▼
Docker Engine
  │
  ├── Pull/Push Images ↔ Docker Registry
  │
  ├── Create Images
  ├── Create Containers
  ├── Create Networks
  └── Create Volumes
```

## Example

- If we run:    ```
                docker run nginx
              ```

Step 1: Docker Client sends the command.

Step 2: Docker Engine receives the request.

Step 3: Docker Engine checks whether the nginx image exists locally.

Step 4: If not available, it pulls the image from Docker Hub.

Step 5: Docker Engine creates a container from the image.

Step 6: The container starts and the application begins running.

- This entire process happens automatically through Docker Architecture.
